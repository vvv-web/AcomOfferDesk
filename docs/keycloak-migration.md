# Keycloak Migration (Актуальная сводка)

## Статус

Миграция аутентификации на Keycloak завершена.
Документ обновлен как краткая сводка текущего состояния системы без исторических переходных деталей.

## Текущее состояние аутентификации

- Web-вход выполняется через Keycloak (OIDC Authorization Code + PKCE).
- Backend проверяет Keycloak JWT и связывает пользователя с локальной учетной записью через `user_auth_accounts`.
- Локальная БД остается источником бизнес-ролей, статусов доступа и иерархии.
- Регистрация контрагентов через внешние ссылки и вход сотрудников работают в едином потоке через страницу `/login`.

## Основные пользовательские сценарии

1. Сотрудник входит через Keycloak и получает доступ по локальному статусу профиля.
2. Контрагент регистрируется по ссылке и после проверки работает по статусу `active`.
3. Контрагент, созданный вручную, задает пароль через `Forgot password` на форме Keycloak и входит стандартным способом.
