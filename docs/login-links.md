# Вход по ссылкам из Telegram и почты

## Назначение

Этот документ фиксирует поведение ссылок, которые пользователь получает из Telegram и email-уведомлений.

## Правило

- Ссылка из Telegram или email не должна создавать web-сессию автоматически.
- Пользователь должен явно открыть стандартную страницу входа и сам нажать вход через Keycloak.
- После успешного входа приложение может вернуть пользователя на целевой экран через параметр `next`.

## Что изменено

### Telegram

- Кнопки в Telegram-уведомлениях больше не ведут в direct auth flow.
- Старый маршрут `/api/v1/tg/auth` теперь только переводит пользователя на обычную страницу `/login`.
- Frontend-страница `/auth/tg/login` больше не обменивает токен на сессию и не обходит форму Keycloak.
- Backend endpoint `/api/v1/auth/tg/exchange` больше не выдаёт сессию и возвращает запрет на direct authentication.

### Email

- Ссылки из email-уведомлений по заявкам ведут на `/login?next=...`.
- После входа через Keycloak пользователь возвращается на нужную страницу заявки.
- Прямой magic-login из письма не используется.

## Практический результат

- Нельзя войти в систему одним кликом по ссылке из Telegram.
- Нельзя войти в систему одним кликом по ссылке из email.
- Для всех внешних ссылок действует единый и явный вход через Keycloak.
