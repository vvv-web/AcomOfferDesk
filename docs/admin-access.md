# Административный доступ

Служебные интерфейсы намеренно отделены от публичного пользовательского пути.

## Политика

- Служебные UI не являются публичными интернет-эндпоинтами.
- Администратор получает к ним доступ только через терминальный сервер.
- Обычные пользователи и контрагенты не работают с admin UI напрямую.

## Admin-only интерфейсы

| Интерфейс | Порт по умолчанию | Статус в текущем compose |
|---|---|---|
| RabbitMQ Management UI | `15672` | Публикуется только в dev |
| MinIO Console | `9001` | Публикуется только в dev |
| pgAdmin | `5050` | Сейчас отсутствует в compose, но если появится, должен оставаться admin-only |
| Прямой доступ к PostgreSQL | `5432` | Внешний сервис, никогда не должен быть публичным |
| Административные возможности Keycloak | проксируются под `/iam` | Ограничиваются административной политикой и IAM-ролями |

## Рекомендуемый путь доступа в test и prod

| Шаг | Путь |
|---|---|
| 1 | Администратор подключается к терминальному серверу |
| 2 | Администратор открывает служебный UI из этого приватного окружения |
| 3 | Приватная маршрутизация доводит запрос до внутреннего/admin endpoint |

## Операционные замечания

- Не публикуйте RabbitMQ UI, MinIO Console, pgAdmin или PostgreSQL в публичный интернет.
- Не используйте `ngrok` и другие публичные tunnel-решения для admin-поверхностей в `test` или `prod`.
- Если admin UI должен быть доступен с deployment host, доступ к нему всё равно должен идти только через терминальный сервер.
