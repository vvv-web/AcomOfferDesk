### Проблема 1
Есть разные объекты с одинаковыми параметрами, но разными типами параметров.
```ts
stringNumberPair -> key: string, value: number
numberArrayPair -> key: number, value: string[]
```
Цель: сделать переиспользуемый тип, поддерживающий оба объекта.
### Решение
Не использовать any. Использовать Generic Type.
```ts
type KeyValuePair<K, V> = {
	key: K;
	value: V;
};

const stringNumberPair: KeyValuePair<string, number> = {
	key: "age",
	value: 30,
};

const numberArrayPair: KeyValuePair<number, string[]> = {
	key: 123,
	value: ["a", "b"],
};
```
### Проблема 2
Нужна функция, которая работает только с объектами, имеющими id: number.
### Решение
Использовать Constraints.
```ts
type HasId = {
	id: number;
};

function printId<T extends HasId>(obj: T): void {
	console.log(obj.id);
}

printId({ id: 3, name: "Alice" }); // ok
printId({ name: "Alice" }); // error
```
